coroutine, gevent, greelet, asyncio, epoll

2015/05/20

coroutine, gevent, greelet, asyncio, epoll

coroutine实现非阻塞原理：在运行耗时的函数时，在每一步用yield跳出，然后别的coroutine执行再跳回耗时的函数。

gevent使用了greelet，例子

from gevent import monkey; monkey.patch_all()
import gevent
import urllib2

def f(url):
    print('GET: %s' % url)
    resp = urllib2.urlopen(url)
    data = resp.read()
    print('%d bytes received from %s.' % (len(data), url))

gevent.joinall([
        gevent.spawn(f, 'https://www.python.org/'),
        gevent.spawn(f, 'https://www.yahoo.com/'),
        gevent.spawn(f, 'https://github.com/'),
])

运行结果：

GET: https://www.python.org/
GET: https://www.yahoo.com/
GET: https://github.com/
47103 bytes received from https://www.python.org/.
279031 bytes received from https://www.yahoo.com/.
17763 bytes received from https://github.com/.

不知道怎么实现无阻塞执行、自动调度的。

asyncio只能在python3中用

epoll： 1. 注册一堆文件描述符和期望描述符发生的事件，暂且称这“一堆”为evenets 2. 程序在无限循环中调用epoll.wait等待事件的发生 3. epoll 将发生事件的文件描述符放入 happend_events中 4. 程序处理happend_events中的文件描述符
